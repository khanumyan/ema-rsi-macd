КРИТЕРИИ СОХРАНЕНИЯ И ФИЛЬТРАЦИИ СИГНАЛОВ

Процесс сохранения сигналов работает в два этапа. Сначала сигналы проходят первичную проверку и сохраняются в базу данных. Затем выполняется финальная фильтрация, которая удаляет из базы сигналы, не соответствующие строгим критериям качества.

═══════════════════════════════════════════════════════════════════════════════

ЭТАП 1: ПЕРВИЧНАЯ ПРОВЕРКА (КРИТЕРИИ ДЛЯ СОХРАНЕНИЯ В БАЗУ)

Сигнал сохраняется в базу данных только если выполняются ВСЕ следующие условия одновременно:

КРИТЕРИЙ 1: ПРОВЕРКА ВЕРОЯТНОСТЕЙ

Условие: Одна из вероятностей должна быть равна 100%

long_probability == 100 ИЛИ short_probability == 100

Объяснение:
- Если long_probability = 100%, это означает, что сигнал имеет 100% вероятность быть BUY сигналом
- Если short_probability = 100%, это означает, что сигнал имеет 100% вероятность быть SELL сигналом
- Если обе вероятности меньше 100%, сигнал считается неопределенным и не сохраняется

Примеры:
✅ long_probability = 100%, short_probability = 0% → проходит проверку
✅ long_probability = 0%, short_probability = 100% → проходит проверку
❌ long_probability = 75%, short_probability = 25% → НЕ проходит (ни одна не равна 100%)
❌ long_probability = 60%, short_probability = 40% → НЕ проходит (ни одна не равна 100%)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 2: ИСКЛЮЧЕНИЕ HOLD СИГНАЛОВ

Условие: Тип сигнала не должен быть HOLD

type != 'HOLD'

Объяснение:
- HOLD означает отсутствие четкого направления (неопределенность)
- Сохраняются только сигналы с четким направлением: BUY или SELL

Примеры:
✅ type = 'BUY' → проходит проверку
✅ type = 'SELL' → проходит проверку
❌ type = 'HOLD' → НЕ проходит проверку

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 3: ЗАЩИТА ОТ ДУБЛИКАТОВ

Условие: Не должно быть дубликата сигнала за последние 30 минут

Для того же символа (symbol) и того же типа (type) не должно существовать сигнала, созданного менее 30 минут назад.

Проверка:
- Ищем в базе данных сигнал где:
  * symbol == текущий символ
  * type == текущий тип (BUY или SELL)
  * created_at >= (текущее время - 30 минут)

Если такой сигнал найден → новый сигнал НЕ сохраняется (дубликат)

Объяснение:
- Предотвращает спам одинаковыми сигналами
- Если для BTCUSDT уже есть BUY сигнал, созданный 15 минут назад, новый BUY для BTCUSDT не будет сохранен
- Через 30 минут можно сохранить новый сигнал того же типа

Примеры:
✅ Первый BUY для BTCUSDT в 10:00 → сохраняется
❌ Второй BUY для BTCUSDT в 10:15 → НЕ сохраняется (прошло только 15 минут)
✅ Третий BUY для BTCUSDT в 10:35 → сохраняется (прошло 35 минут)
✅ SELL для BTCUSDT в 10:20 → сохраняется (другой тип, не дубликат)

───────────────────────────────────────────────────────────────────────────────

ИТОГ ПЕРВИЧНОЙ ПРОВЕРКИ

Если сигнал проходит ВСЕ три критерия:
✅ long_probability == 100 ИЛИ short_probability == 100
✅ type != 'HOLD'
✅ Нет дубликата за последние 30 минут

То сигнал сохраняется в базу данных со всеми данными:
- Все индикаторы (RSI, EMA, MACD, ATR)
- Все баллы и вероятности
- Уровни Stop Loss и Take Profit
- Метаданные (символ, стратегия, причина сигнала и т.д.)

═══════════════════════════════════════════════════════════════════════════════

ЭТАП 2: ФИНАЛЬНАЯ ФИЛЬТРАЦИЯ (КРИТЕРИИ ЧТОБЫ НЕ УДАЛИЛИ ИЗ БАЗЫ)

После сохранения всех сигналов потока выполняется строгая фильтрация. Сигналы, не прошедшие проверку, УДАЛЯЮТСЯ из базы данных.

ОБЩИЕ КРИТЕРИИ (для всех сигналов BUY и SELL)

Все сигналы должны соответствовать следующим условиям:

КРИТЕРИЙ 1: НАЛИЧИЕ ОБЯЗАТЕЛЬНЫХ ПОЛЕЙ

Все поля должны быть заполнены (не NULL):

atr IS NOT NULL
ema IS NOT NULL
macd_histogram IS NOT NULL
take_profit IS NOT NULL

Объяснение: Все ключевые индикаторы должны быть успешно рассчитаны. Если какой-то индикатор не рассчитан, сигнал удаляется.

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 2: ВАЛИДНОСТЬ ЦЕНОВЫХ ДАННЫХ

price > 0
atr > 0

Объяснение: Цена и волатильность должны быть положительными числами. Нулевые или отрицательные значения недопустимы.

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 3: ФИЛЬТР РАССТОЯНИЯ TAKE PROFIT

(ABS((price - take_profit) / price) * 100) <= 3

Формула: |цена - take_profit| / цена × 100 ≤ 3%

Объяснение:
- Расстояние от текущей цены до уровня Take Profit не должно превышать 3% от цены
- Это гарантирует, что TP находится в разумных пределах
- Слишком далекий TP может указывать на ошибку в расчетах

Примеры:
✅ Цена = $50,000, Take Profit = $51,400
   Расстояние = |50000 - 51400| / 50000 × 100 = 2.8% → проходит

❌ Цена = $50,000, Take Profit = $52,000
   Расстояние = |50000 - 52000| / 50000 × 100 = 4.0% → НЕ проходит

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИИ ДЛЯ BUY СИГНАЛОВ

BUY сигнал остается в базе только если выполняются ВСЕ следующие условия одновременно:

КРИТЕРИЙ 1: RSI В ОПТИМАЛЬНОЙ ЗОНЕ ДЛЯ ПОКУПКИ

rsi > 48 AND rsi < 60

Объяснение:
- RSI должен быть в диапазоне от 48 до 60
- Это зона умеренной перепроданности, оптимальная для входа в лонг
- RSI < 48 → слишком перепродан, возможен отскок, но нестабильно
- RSI > 60 → перекуплен, не подходит для покупки

Примеры:
✅ RSI = 52 → проходит (48 < 52 < 60)
✅ RSI = 50 → проходит (48 < 50 < 60)
❌ RSI = 45 → НЕ проходит (45 < 48)
❌ RSI = 65 → НЕ проходит (65 > 60)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 2: MACD HISTOGRAM НОРМАЛИЗОВАННЫЙ ЧЕРЕЗ ATR

(macd_histogram / atr) >= 0.25

Формула: macd_histogram / atr ≥ 0.25

Объяснение:
- MACD Histogram должен быть не менее 25% от ATR
- Это подтверждает наличие достаточного импульса для покупки
- Нормализация через ATR делает критерий универсальным для всех токенов

Примеры:
✅ MACD Histogram = 150, ATR = 500
   Отношение = 150 / 500 = 0.30 → проходит (0.30 ≥ 0.25)

❌ MACD Histogram = 100, ATR = 800
   Отношение = 100 / 800 = 0.125 → НЕ проходит (0.125 < 0.25)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 3: РАССТОЯНИЕ ЦЕНЫ ОТ EMA НОРМАЛИЗОВАННОЕ ЧЕРЕЗ ATR

(ABS(price - ema) / atr) BETWEEN 0.5 AND 1.5

Формула: |цена - ema| / atr должно быть между 0.5 и 1.5

Объяснение:
- Расстояние от цены до EMA должно быть от 0.5 до 1.5 ATR
- < 0.5 ATR → цена слишком близко к EMA, флет, нет движения
- > 1.5 ATR → цена слишком далеко от EMA, поздно входить, возможна коррекция
- 0.5-1.5 ATR → оптимальная зона для входа

Примеры:
✅ Цена = $50,000, EMA = $49,500, ATR = $500
   Расстояние = |50000 - 49500| / 500 = 1.0 → проходит (0.5 ≤ 1.0 ≤ 1.5)

❌ Цена = $50,000, EMA = $50,200, ATR = $500
   Расстояние = |50000 - 50200| / 500 = 0.4 → НЕ проходит (0.4 < 0.5)

❌ Цена = $50,000, EMA = $48,000, ATR = $500
   Расстояние = |50000 - 48000| / 500 = 4.0 → НЕ проходит (4.0 > 1.5)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 4: ВОЛАТИЛЬНОСТЬ В ДОПУСТИМЫХ ПРЕДЕЛАХ

((atr / price) * 100) BETWEEN 0.3 AND 3.0

Формула: (atr / цена) × 100 должно быть между 0.3% и 3.0%

Объяснение:
- ATR в процентах от цены должен быть от 0.3% до 3.0%
- < 0.3% → мертвый рынок, нет движения, неинтересно
- > 3.0% → слишком волатильный рынок, высокий риск
- 0.3-3.0% → нормальная волатильность для торговли

Примеры:
✅ Цена = $50,000, ATR = $500
   ATR% = (500 / 50000) × 100 = 1.0% → проходит (0.3 ≤ 1.0 ≤ 3.0)

❌ Цена = $50,000, ATR = $100
   ATR% = (100 / 50000) × 100 = 0.2% → НЕ проходит (0.2 < 0.3)

❌ Цена = $50,000, ATR = $2,000
   ATR% = (2000 / 50000) × 100 = 4.0% → НЕ проходит (4.0 > 3.0)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 5: РАЗНИЦА БАЛЛОВ МЕЖДУ BUY И SELL

(long_score - short_score) BETWEEN 10 AND 20

Формула: long_score - short_score должно быть между 10 и 20

Объяснение:
- Разница между баллами BUY и SELL должна быть от 10 до 20
- < 10 → недостаточное превосходство BUY над SELL, неопределенность
- > 20 → слишком экстремальное превосходство, возможен перегрев
- 10-20 → оптимальная разница, подтверждает силу сигнала

Примеры:
✅ long_score = 65, short_score = 50
   Разница = 65 - 50 = 15 → проходит (10 ≤ 15 ≤ 20)

❌ long_score = 60, short_score = 55
   Разница = 60 - 55 = 5 → НЕ проходит (5 < 10)

❌ long_score = 80, short_score = 40
   Разница = 80 - 40 = 40 → НЕ проходит (40 > 20)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИИ ДЛЯ SELL СИГНАЛОВ

SELL сигнал остается в базе только если выполняются ВСЕ следующие условия одновременно:

КРИТЕРИЙ 1: RSI В ОПТИМАЛЬНОЙ ЗОНЕ ДЛЯ ПРОДАЖИ

rsi BETWEEN 40 AND 52

Объяснение:
- RSI должен быть в диапазоне от 40 до 52
- Это зона умеренной перекупленности, оптимальная для входа в шорт
- RSI < 40 → слишком перепродан, не подходит для продажи
- RSI > 52 → недостаточно перекуплен для продажи

Примеры:
✅ RSI = 48 → проходит (40 ≤ 48 ≤ 52)
✅ RSI = 45 → проходит (40 ≤ 45 ≤ 52)
❌ RSI = 35 → НЕ проходит (35 < 40)
❌ RSI = 55 → НЕ проходит (55 > 52)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 2: MACD HISTOGRAM НОРМАЛИЗОВАННЫЙ ЧЕРЕЗ ATR (ДЛЯ SELL)

(ABS(macd_histogram) / atr) >= 0.25

Формула: |macd_histogram| / atr ≥ 0.25

Объяснение:
- Абсолютное значение MACD Histogram должно быть не менее 25% от ATR
- Для SELL используется ABS(), т.к. histogram отрицательный при медвежьем тренде
- Подтверждает наличие достаточного импульса для продажи

Примеры:
✅ MACD Histogram = -200, ATR = 500
   Отношение = |-200| / 500 = 0.40 → проходит (0.40 ≥ 0.25)

❌ MACD Histogram = -150, ATR = 1000
   Отношение = |-150| / 1000 = 0.15 → НЕ проходит (0.15 < 0.25)

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 3: РАССТОЯНИЕ ЦЕНЫ ОТ EMA НОРМАЛИЗОВАННОЕ ЧЕРЕЗ ATR

(ABS(price - ema) / atr) BETWEEN 0.5 AND 1.5

Формула: |цена - ema| / atr должно быть между 0.5 и 1.5

Объяснение: Аналогично BUY сигналам - оптимальная зона для входа в шорт

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 4: ВОЛАТИЛЬНОСТЬ В ДОПУСТИМЫХ ПРЕДЕЛАХ

((atr / price) * 100) BETWEEN 0.3 AND 3.0

Формула: (atr / цена) × 100 должно быть между 0.3% и 3.0%

Объяснение: Аналогично BUY сигналам - нормальная волатильность для торговли

───────────────────────────────────────────────────────────────────────────────

КРИТЕРИЙ 5: РАЗНИЦА БАЛЛОВ МЕЖДУ SELL И BUY

(short_score - long_score) BETWEEN 10 AND 20

Формула: short_score - long_score должно быть между 10 и 20

Объяснение:
- Разница между баллами SELL и BUY должна быть от 10 до 20
- Подтверждает превосходство SELL сигнала над BUY

Примеры:
✅ short_score = 70, long_score = 55
   Разница = 70 - 55 = 15 → проходит (10 ≤ 15 ≤ 20)

❌ short_score = 60, long_score = 58
   Разница = 60 - 58 = 2 → НЕ проходит (2 < 10)

❌ short_score = 85, long_score = 50
   Разница = 85 - 50 = 35 → НЕ проходит (35 > 20)

ИТОГОВАЯ ЛОГИКА ФИЛЬТРАЦИИ

ПРОЦЕСС РАБОТЫ:

1. Сохранение сигналов потока:
   - Применяются первичные фильтры (вероятности, дубликаты, HOLD)
   - Все прошедшие сигналы сохраняются в базу с flow_id

2. Финальная фильтрация:
   - Выбираются сигналы текущего потока (по flow_id)
   - Применяются строгие критерии (общие + специфичные для BUY/SELL)
   - Сигналы, не прошедшие проверку, УДАЛЯЮТСЯ из базы

3. Результат:
   - В базе остаются только сигналы, прошедшие ВСЕ критерии
   - Если ни один сигнал не прошел фильтр → все сигналы потока удаляются

ПРИМЕР РАБОТЫ:

Сценарий: BUY сигнал для BTCUSDT

ПЕРВИЧНАЯ ПРОВЕРКА:
✅ long_probability = 100% → проходит
✅ type = BUY (не HOLD) → проходит
✅ Нет дубликата за 30 минут → проходит
→ Сигнал сохраняется в базу

ФИНАЛЬНАЯ ФИЛЬТРАЦИЯ:
✅ Общие критерии: все поля заполнены, цены валидны
✅ TP distance: 2.8% ≤ 3% → проходит
✅ BUY критерии:
   ✅ RSI: 52 (48 < 52 < 60) → проходит
   ✅ MACD/ATR: 0.30 ≥ 0.25 → проходит
   ✅ EMA distance: 1.0 (0.5 ≤ 1.0 ≤ 1.5) → проходит
   ✅ ATR%: 1.0% (0.3 ≤ 1.0 ≤ 3.0) → проходит
   ✅ Score diff: 15 (10 ≤ 15 ≤ 20) → проходит

→ Сигнал ОСТАЕТСЯ в базе данных

КРАТКАЯ СВОДКА КРИТЕРИЕВ

ПЕРВИЧНАЯ ПРОВЕРКА (для сохранения):
1. long_probability == 100 ИЛИ short_probability == 100
2. type != 'HOLD'
3. Нет дубликата за последние 30 минут

ФИНАЛЬНАЯ ФИЛЬТРАЦИЯ (чтобы не удалили):

ОБЩИЕ (для всех):
- atr, ema, macd_histogram, take_profit не NULL
- price > 0, atr > 0
- |цена - take_profit| / цена × 100 ≤ 3%

ДЛЯ BUY:
- rsi > 48 AND rsi < 60
- macd_histogram / atr ≥ 0.25
- |цена - ema| / atr BETWEEN 0.5 AND 1.5
- (atr / цена) × 100 BETWEEN 0.3 AND 3.0
- (long_score - short_score) BETWEEN 10 AND 20

ДЛЯ SELL:
- rsi BETWEEN 40 AND 52
- |macd_histogram| / atr ≥ 0.25
- |цена - ema| / atr BETWEEN 0.5 AND 1.5
- (atr / цена) × 100 BETWEEN 0.3 AND 3.0
- (short_score - long_score) BETWEEN 10 AND 20
