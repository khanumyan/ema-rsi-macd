ПРОЦЕСС СБОРА ДАННЫХ И АНАЛИЗА КРИПТОВАЛЮТ

Каждые 15 минут наша система автоматически выполняет запрос к Binance Futures API для получения актуальных данных о криптовалютах. Для каждого токена из списка мы запрашиваем исторические данные свечей и проводим технический анализ.

ЗАПРОС ДАННЫХ ОТ BINANCE

Мы используем публичный API Binance Futures (fapi.binance.com) без необходимости аутентификации. Для каждого токена отправляется HTTP GET запрос к эндпоинту:

URL: https://fapi.binance.com/fapi/v1/klines

Параметры запроса:
- symbol: символ токена с суффиксом USDT (например, BTCUSDT, ETHUSDT, BNBUSDT)
- interval: таймфрейм свечей (по умолчанию 15m - 15 минут)
- limit: количество свечей (по умолчанию 200 свечей)

Пример запроса для BTC:
https://fapi.binance.com/fapi/v1/klines?symbol=BTCUSDT&interval=15m&limit=200

СТРУКТУРА ОТВЕТА ОТ API BINANCE

API возвращает массив массивов, где каждая свеча представлена массивом из 12 элементов:

[0] - Open time (время открытия свечи в миллисекундах)
[1] - Open price (цена открытия свечи)
[2] - High price (максимальная цена за период свечи)
[3] - Low price (минимальная цена за период свечи)
[4] - Close price (цена закрытия свечи) - ОСНОВНАЯ ЦЕНА ДЛЯ АНАЛИЗА
[5] - Volume (объем торгов в базовой валюте)
[6] - Close time (время закрытия свечи в миллисекундах)
[7] - Quote asset volume (объем торгов в котируемой валюте USDT)
[8] - Number of trades (количество сделок)
[9] - Taker buy base asset volume (объем покупок маркет-мейкеров)
[10] - Taker buy quote asset volume (объем покупок в USDT)
[11] - Ignore (не используется)

Для нашего анализа мы используем следующие данные из каждой свечи:
- Close price [4] - цена закрытия (основная цена для всех расчетов)
- High price [2] - максимальная цена (для расчета ATR)
- Low price [3] - минимальная цена (для расчета ATR)

Из 200 свечей мы извлекаем три массива:
- Массив цен закрытия (closes) - 200 значений
- Массив максимальных цен (highs) - 200 значений
- Массив минимальных цен (lows) - 200 значений

Текущая цена токена берется как последнее значение из массива цен закрытия (последняя закрытая свеча).

РАСЧЕТ ТЕХНИЧЕСКИХ ИНДИКАТОРОВ

После получения данных от Binance мы рассчитываем следующие технические индикаторы:

1. EMA (Exponential Moving Average) - Экспоненциальная скользящая средняя

Мы рассчитываем две EMA:
- EMA(20) - быстрая скользящая средняя с периодом 20 свечей
- EMA(50) - медленная скользящая средняя с периодом 50 свечей

Формула расчета EMA:
EMA = (Текущая цена × Множитель) + (Предыдущая EMA × (1 - Множитель))
где Множитель = 2 / (Период + 1)

Для EMA(20): Множитель = 2 / 21 = 0.0952
Для EMA(50): Множитель = 2 / 51 = 0.0392

Начальное значение EMA рассчитывается как простая средняя арифметическая (SMA) первых N свечей, где N - период EMA.

2. RSI (Relative Strength Index) - Индекс относительной силы

RSI рассчитывается с периодом 14 свечей.

Алгоритм расчета:
- Вычисляем изменения цены между соседними свечами (deltas)
- Разделяем изменения на gains (приросты, положительные значения) и losses (убытки, отрицательные значения, берем по модулю)
- Рассчитываем средние значения gains и losses за 14 периодов
- Применяем сглаживание по методу Уайлдера (Wilder's smoothing)
- Вычисляем Relative Strength (RS) = Средний Gain / Средний Loss
- RSI = 100 - (100 / (1 + RS))

RSI находится в диапазоне от 0 до 100:
- RSI < 30 - перепроданность (потенциал для роста)
- RSI > 70 - перекупленность (потенциал для падения)
- RSI 40-60 - нейтральная зона

3. MACD (Moving Average Convergence Divergence) - Схождение-расхождение скользящих средних

MACD состоит из трех компонентов:

MACD Line (линия MACD):
- Рассчитывается как разница между EMA(12) и EMA(26)
- MACD Line = EMA(12) - EMA(26)

Signal Line (сигнальная линия):
- Рассчитывается как EMA(9) от значений MACD Line
- Для этого сначала вычисляются исторические значения MACD Line для каждой свечи
- Затем к этим значениям применяется EMA(9)

Histogram (гистограмма):
- Рассчитывается как разница между MACD Line и Signal Line
- Histogram = MACD Line - Signal Line

Интерпретация:
- MACD Line > 0 и Histogram > 0 - бычий сигнал
- MACD Line < 0 и Histogram < 0 - медвежий сигнал
- Пересечение MACD Line и Signal Line - сигнал на покупку/продажу

4. ATR (Average True Range) - Средний истинный диапазон

ATR рассчитывается с периодом 14 свечей и показывает волатильность рынка.

Алгоритм расчета:
- Для каждой свечи вычисляем True Range (TR):
  TR = максимум из трех значений:
  1. High - Low (текущая свеча)
  2. |High - Previous Close| (абсолютное значение)
  3. |Low - Previous Close| (абсолютное значение)

- ATR = среднее арифметическое значений True Range за 14 периодов
- Применяется сглаживание по методу Уайлдера

ATR используется для:
- Нормализации других индикаторов (делает критерии универсальными для всех токенов)
- Расчет уровней Stop Loss и Take Profit

АНАЛИТИКА И ГЕНЕРАЦИЯ СИГНАЛОВ

После расчета всех индикаторов начинается процесс аналитики, который работает по балльной системе.

ИНИЦИАЛИЗАЦИЯ БАЛЛОВ

Для каждого токена инициализируются два счетчика:
- long_score = 0 (баллы для сигнала BUY/покупки)
- short_score = 0 (баллы для сигнала SELL/продажи)

РАСЧЕТ БАЛЛОВ ДЛЯ BUY СИГНАЛОВ

1. Основное условие (40 баллов):
Если выполняются ВСЕ условия одновременно:
- Цена > EMA(20) (цена выше быстрой скользящей средней)
- EMA(20) > EMA(50) (быстрая средняя выше медленной - бычий тренд)
- MACD Line > 0 (MACD выше нуля - бычий импульс)
- MACD Histogram > 0 (гистограмма положительная - усиление импульса)
- RSI < 70 (не перекуплен)

То добавляется 40 базовых баллов.

Дополнительные баллы за импульс:
- Если RSI находится в диапазоне 40-60 И абсолютное значение Histogram > 0.5: +30 баллов (сильный импульс)
- Иначе, если RSI > 30 И абсолютное значение Histogram > 0.2: +15 баллов (средний импульс)

2. Альтернативное условие (20 баллов):
Если цена > EMA(20) И EMA(20) > EMA(50), но не выполняются условия MACD: +20 баллов

3. Условие перепроданности (20 баллов):
Если RSI ≤ 30 (сильная перепроданность - потенциал отскока вверх): +20 баллов

4. Бычье пересечение MACD (10 баллов):
Если MACD Histogram > 0 И MACD Line > Signal Line (бычье пересечение): +10 баллов

РАСЧЕТ БАЛЛОВ ДЛЯ SELL СИГНАЛОВ

1. Основное условие (40 баллов):
Если выполняются ВСЕ условия одновременно:
- Цена < EMA(20) (цена ниже быстрой скользящей средней)
- EMA(20) < EMA(50) (быстрая средняя ниже медленной - медвежий тренд)
- MACD Line < 0 (MACD ниже нуля - медвежий импульс)
- MACD Histogram < 0 (гистограмма отрицательная - усиление импульса)
- RSI > 30 (не перепродан)

То добавляется 40 базовых баллов.

Дополнительные баллы за импульс:
- Если RSI находится в диапазоне 40-60 И абсолютное значение Histogram > 0.5: +30 баллов (сильный импульс)
- Иначе, если RSI < 70 И абсолютное значение Histogram > 0.2: +15 баллов (средний импульс)

2. Альтернативное условие (20 баллов):
Если цена < EMA(20) И EMA(20) < EMA(50), но не выполняются условия MACD: +20 баллов

3. Условие перекупленности (20 баллов):
Если RSI ≥ 70 (сильная перекупленность - потенциал падения): +20 баллов

4. Медвежье пересечение MACD (10 баллов):
Если MACD Histogram < 0 И MACD Line < Signal Line (медвежье пересечение): +10 баллов

ПРЕОБРАЗОВАНИЕ БАЛЛОВ В ВЕРОЯТНОСТИ

После расчета всех баллов они преобразуются в вероятности:

total_score = long_score + short_score

Если total_score > 0:
- long_probability = (long_score / total_score) × 100 (вероятность BUY в процентах)
- short_probability = (short_score / total_score) × 100 (вероятность SELL в процентах)

Если total_score = 0 (оба счетчика равны нулю):
- long_probability = 50%
- short_probability = 50%

ОПРЕДЕЛЕНИЕ ТИПА СИГНАЛА

На основе вероятностей определяется тип сигнала:

- BUY: если long_probability > short_probability И long_probability > 50%
- SELL: если short_probability > long_probability И short_probability > 50%
- HOLD: во всех остальных случаях (когда разница незначительна или вероятности близки к 50%)

ОПРЕДЕЛЕНИЕ СИЛЫ СИГНАЛА

Сила сигнала определяется по разнице между вероятностями:

prob_difference = |long_probability - short_probability|

- STRONG (сильный): если prob_difference > 20%
- MEDIUM (средний): если prob_difference > 10%
- WEAK (слабый): если prob_difference ≤ 10%

РАСЧЕТ УРОВНЕЙ STOP LOSS И TAKE PROFIT

Для каждого сигнала BUY или SELL рассчитываются уровни входа и выхода:

Для BUY сигнала:
- Stop Loss = цена - (ATR × 2.3)
- Take Profit = цена + (ATR × 2.0)

Для SELL сигнала:
- Stop Loss = цена + (ATR × 2.3)
- Take Profit = цена - (ATR × 2.0)

Коэффициенты 2.3 и 2.0 подобраны эмпирически для оптимального соотношения риск/прибыль.

ФОРМИРОВАНИЕ РЕЗУЛЬТАТА АНАЛИЗА

Для каждого токена формируется массив результатов, содержащий:

- type: тип сигнала (BUY/SELL/HOLD)
- strength: сила сигнала (STRONG/MEDIUM/WEAK)
- price: текущая цена токена
- rsi: значение RSI(14)
- ema: значение EMA(20)
- ema_slow: значение EMA(50)
- macd: значение MACD Line
- macd_signal: значение Signal Line
- macd_histogram: значение Histogram
- atr: значение ATR(14)
- stop_loss: уровень стоп-лосса
- take_profit: уровень тейк-профита
- long_score: сумма баллов для BUY
- short_score: сумма баллов для SELL
- long_probability: вероятность BUY в процентах
- short_probability: вероятность SELL в процентах
- reason: текстовое описание причины сигнала

Все эти данные сохраняются в базу данных для дальнейшего анализа и применения дополнительных фильтров качества сигналов.
