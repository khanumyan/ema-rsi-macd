# Логика алгоритма сохранения сигналов EMA+RSI+MACD

## Обзор процесса

Алгоритм сохранения сигналов работает в **два этапа**:

1. **Первичное сохранение** - фильтрация по базовым критериям и сохранение в базу данных
2. **Финальная фильтрация потока** - применение строгих критериев и удаление сигналов, не прошедших проверку

---

## Этап 1: Первичное сохранение сигналов

### Критерий 1: Проверка вероятностей

**Условие:** Сигнал сохраняется только если **одна из вероятностей равна 100%**

```php
if (long_probability == 100 OR short_probability == 100) {
    // Продолжаем проверку
} else {
    // Сигнал НЕ сохраняется
    return false;
}
```

**Логика:**
- `long_probability = 100%` → сигнал имеет 100% вероятность быть BUY
- `short_probability = 100%` → сигнал имеет 100% вероятность быть SELL
- Если обе вероятности меньше 100% → сигнал неопределенный, не сохраняем

**Пример:**
- ✅ `long_probability = 100%`, `short_probability = 0%` → сохраняется
- ✅ `long_probability = 0%`, `short_probability = 100%` → сохраняется
- ❌ `long_probability = 75%`, `short_probability = 25%` → НЕ сохраняется

---

### Критерий 2: Исключение HOLD сигналов

**Условие:** Сигналы типа HOLD не сохраняются

```php
if (type == 'HOLD') {
    return false; // Не сохраняем HOLD сигналы
}
```

**Логика:**
- HOLD означает отсутствие четкого направления
- Сохраняем только BUY и SELL сигналы

---

### Критерий 3: Защита от дубликатов

**Условие:** Не сохраняем дубликаты сигналов за последние 30 минут

```php
$cutoffTime = now() - 30 минут;
$existingSignal = найти сигнал где:
    - symbol == текущий символ
    - type == текущий тип (BUY/SELL)
    - created_at >= cutoffTime

if ($existingSignal существует) {
    return false; // Дубликат, не сохраняем
}
```

**Логика:**
- Если для символа BTCUSDT уже есть BUY сигнал, созданный менее 30 минут назад
- Новый BUY сигнал для BTCUSDT не будет сохранен
- Это предотвращает спам одинаковыми сигналами

**Пример:**
- ✅ Первый BUY для BTCUSDT в 10:00 → сохраняется
- ❌ Второй BUY для BTCUSDT в 10:15 → НЕ сохраняется (дубликат)
- ✅ Третий BUY для BTCUSDT в 10:35 → сохраняется (прошло 35 минут)

---

### Критерий 4: Сохранение всех данных

Если сигнал прошел все предыдущие проверки, сохраняются **ВСЕ данные**:

**Основные данные:**
- `flow_id` - уникальный идентификатор потока анализа
- `symbol` - символ криптовалюты (BTCUSDT, ETHUSDT и т.д.)
- `strategy` - название стратегии ("EMA+RSI+MACD")
- `type` - тип сигнала (BUY/SELL)
- `strength` - сила сигнала (STRONG/MEDIUM/WEAK)
- `price` - текущая цена закрытия

**Технические индикаторы:**
- `rsi` - RSI(14) - индекс относительной силы
- `ema` - EMA(20) - быстрая экспоненциальная скользящая средняя
- `ema_slow` - EMA(50) - медленная экспоненциальная скользящая средняя
- `macd` - MACD Line = EMA(12) - EMA(26)
- `macd_signal` - MACD Signal Line = EMA(9) от MACD
- `macd_histogram` - MACD Histogram = MACD - Signal
- `atr` - ATR(14) - средний истинный диапазон

**Уровни входа/выхода:**
- `stop_loss` - уровень стоп-лосса
- `take_profit` - уровень тейк-профита

**Балльная система:**
- `long_score` - сумма баллов для BUY (0-100)
- `short_score` - сумма баллов для SELL (0-100)
- `long_probability` - вероятность BUY в процентах
- `short_probability` - вероятность SELL в процентах

**Параметры запроса:**
- `interval` - таймфрейм (15m, 1h, 4h и т.д.)
- `limit` - количество свечей (обычно 200)

**Метаданные:**
- `reason` - текстовое описание причины сигнала
- `sent_to_telegram` - был ли отправлен в Telegram
- `signal_time` - время сигнала (created_at + 4 часа)
- `status` - статус сигнала (устанавливается позже)

---

## Этап 2: Финальная фильтрация потока

После сохранения всех сигналов потока выполняется **строгая фильтрация**. Сигналы, не прошедшие проверку, **удаляются из базы данных**.

### Общие критерии для всех сигналов

Все сигналы (BUY и SELL) должны соответствовать следующим условиям:

#### 1. Наличие обязательных полей

```sql
atr IS NOT NULL
AND ema IS NOT NULL
AND macd_histogram IS NOT NULL
AND take_profit IS NOT NULL
```

**Логика:** Все ключевые индикаторы должны быть рассчитаны

---

#### 2. Валидность ценовых данных

```sql
price > 0
AND atr > 0
```

**Логика:** Цена и волатильность должны быть положительными числами

---

#### 3. Фильтр расстояния Take Profit

```sql
(ABS((price - take_profit) / price) * 100) <= 3
```

**Логика:** 
- Расстояние от цены до Take Profit не должно превышать 3% от цены
- Это гарантирует, что TP находится в разумных пределах
- Формула: `|цена - take_profit| / цена × 100 ≤ 3%`

**Пример:**
- Цена: $50,000
- Take Profit: $51,400
- Расстояние: |50000 - 51400| / 50000 × 100 = 2.8% ✅
- Если бы TP был $52,000 → расстояние = 4% ❌ (не прошло бы)

---

### Критерии для BUY сигналов

BUY сигнал сохраняется только если **ВСЕ** следующие условия выполнены одновременно:

#### 1️⃣ RSI в оптимальной зоне для покупки

```sql
rsi > 48 AND rsi < 60
```

**Логика:**
- RSI должен быть в диапазоне от 48 до 60
- Это зона умеренной перепроданности, оптимальная для входа в лонг
- RSI < 48 → слишком перепродан, возможен отскок, но нестабильно
- RSI > 60 → перекуплен, не подходит для покупки

**Пример:**
- ✅ RSI = 52 → проходит
- ❌ RSI = 45 → не проходит (слишком низко)
- ❌ RSI = 65 → не проходит (слишком высоко)

---

#### 2️⃣ MACD Histogram нормализованный через ATR

```sql
(macd_histogram / atr) >= 0.25
```

**Логика:**
- MACD Histogram должен быть не менее 25% от ATR
- Это подтверждает наличие достаточного импульса для покупки
- Нормализация через ATR делает критерий универсальным для всех коинов

**Формула:** `macd_histogram / atr ≥ 0.25`

**Пример:**
- MACD Histogram = 150
- ATR = 500
- Отношение = 150 / 500 = 0.30 ✅ (проходит, т.к. ≥ 0.25)
- Если бы ATR = 800 → отношение = 0.1875 ❌ (не проходит)

---

#### 3️⃣ Расстояние цены от EMA нормализованное через ATR

```sql
(ABS(price - ema) / atr) BETWEEN 0.5 AND 1.5
```

**Логика:**
- Расстояние от цены до EMA должно быть от 0.5 до 1.5 ATR
- < 0.5 ATR → цена слишком близко к EMA, флет, нет движения
- > 1.5 ATR → цена слишком далеко от EMA, поздно входить, возможна коррекция
- 0.5-1.5 ATR → оптимальная зона для входа

**Формула:** `|цена - ema| / atr` должно быть между 0.5 и 1.5

**Пример:**
- Цена = $50,000
- EMA = $49,500
- ATR = $500
- Расстояние = |50000 - 49500| / 500 = 1.0 ✅ (проходит)
- Если бы EMA = $50,200 → расстояние = 0.4 ❌ (слишком близко)
- Если бы EMA = $48,000 → расстояние = 4.0 ❌ (слишком далеко)

---

#### 4️⃣ Волатильность в допустимых пределах

```sql
((atr / price) * 100) BETWEEN 0.3 AND 3.0
```

**Логика:**
- ATR в процентах от цены должен быть от 0.3% до 3.0%
- < 0.3% → мертвый рынок, нет движения, неинтересно
- > 3.0% → слишком волатильный рынок, высокий риск
- 0.3-3.0% → нормальная волатильность для торговли

**Формула:** `(atr / цена) × 100` должно быть между 0.3% и 3.0%

**Пример:**
- Цена = $50,000
- ATR = $500
- ATR% = (500 / 50000) × 100 = 1.0% ✅ (проходит)
- Если бы ATR = $100 → ATR% = 0.2% ❌ (слишком низкая волатильность)
- Если бы ATR = $2,000 → ATR% = 4.0% ❌ (слишком высокая волатильность)

---

#### 5️⃣ Разница баллов между BUY и SELL

```sql
(long_score - short_score) BETWEEN 10 AND 20
```

**Логика:**
- Разница между баллами BUY и SELL должна быть от 10 до 20
- < 10 → недостаточное превосходство BUY над SELL, неопределенность
- > 20 → слишком экстремальное превосходство, возможен перегрев
- 10-20 → оптимальная разница, подтверждает силу сигнала

**Формула:** `long_score - short_score` должно быть между 10 и 20

**Пример:**
- long_score = 65
- short_score = 50
- Разница = 65 - 50 = 15 ✅ (проходит)
- Если бы short_score = 60 → разница = 5 ❌ (слишком мало)
- Если бы short_score = 40 → разница = 25 ❌ (слишком много)

---

### Критерии для SELL сигналов

SELL сигнал сохраняется только если **ВСЕ** следующие условия выполнены одновременно:

#### 1️⃣ RSI в оптимальной зоне для продажи

```sql
rsi BETWEEN 40 AND 52
```

**Логика:**
- RSI должен быть в диапазоне от 40 до 52
- Это зона умеренной перекупленности, оптимальная для входа в шорт
- RSI < 40 → слишком перепродан, не подходит для продажи
- RSI > 52 → недостаточно перекуплен для продажи

**Пример:**
- ✅ RSI = 48 → проходит
- ❌ RSI = 35 → не проходит (слишком низко)
- ❌ RSI = 55 → не проходит (слишком высоко)

---

#### 2️⃣ MACD Histogram нормализованный через ATR (для SELL)

```sql
(ABS(macd_histogram) / atr) >= 0.25
```

**Логика:**
- Абсолютное значение MACD Histogram должно быть не менее 25% от ATR
- Для SELL используется ABS(), т.к. histogram отрицательный при медвежьем тренде
- Подтверждает наличие достаточного импульса для продажи

**Формула:** `|macd_histogram| / atr ≥ 0.25`

**Пример:**
- MACD Histogram = -200 (отрицательный для SELL)
- ATR = 500
- Отношение = |-200| / 500 = 0.40 ✅ (проходит)
- Если бы ATR = 1000 → отношение = 0.20 ❌ (не проходит)

---

#### 3️⃣ Расстояние цены от EMA нормализованное через ATR

```sql
(ABS(price - ema) / atr) BETWEEN 0.5 AND 1.5
```

**Логика:** Аналогично BUY сигналам
- Расстояние от цены до EMA должно быть от 0.5 до 1.5 ATR
- Оптимальная зона для входа в шорт

---

#### 4️⃣ Волатильность в допустимых пределах

```sql
((atr / price) * 100) BETWEEN 0.3 AND 3.0
```

**Логика:** Аналогично BUY сигналам
- ATR в процентах от цены должен быть от 0.3% до 3.0%

---

#### 5️⃣ Разница баллов между SELL и BUY

```sql
(short_score - long_score) BETWEEN 10 AND 20
```

**Логика:**
- Разница между баллами SELL и BUY должна быть от 10 до 20
- Подтверждает превосходство SELL сигнала над BUY

**Формула:** `short_score - long_score` должно быть между 10 и 20

**Пример:**
- short_score = 70
- long_score = 55
- Разница = 70 - 55 = 15 ✅ (проходит)

---

## Итоговая логика фильтрации

### Процесс работы:

1. **Сохранение сигналов потока:**
   - Применяются первичные фильтры (вероятности, дубликаты, HOLD)
   - Все прошедшие сигналы сохраняются в базу с `flow_id`

2. **Финальная фильтрация:**
   - Выбираются сигналы текущего потока (`flow_id`)
   - Применяются строгие критерии (общие + специфичные для BUY/SELL)
   - Сигналы, не прошедшие проверку, **удаляются из базы**

3. **Результат:**
   - В базе остаются только сигналы, прошедшие **ВСЕ** критерии
   - Если ни один сигнал не прошел фильтр → все сигналы потока удаляются

---

## Преимущества двухэтапной системы

### Этап 1 (Первичное сохранение):
- ✅ Быстрая фильтрация очевидных невалидных сигналов
- ✅ Защита от дубликатов
- ✅ Сохранение только сигналов с 100% вероятностью

### Этап 2 (Финальная фильтрация):
- ✅ Строгие критерии качества сигналов
- ✅ Нормализация через ATR для универсальности
- ✅ Фильтрация по волатильности и расстояниям
- ✅ Проверка баланса между BUY и SELL баллами

---

## Пример работы алгоритма

### Сценарий 1: Сигнал проходит все проверки

**Входные данные:**
- Символ: BTCUSDT
- Тип: BUY
- long_probability: 100%
- short_probability: 0%
- RSI: 52
- MACD Histogram: 150
- ATR: 500
- EMA: 49,500
- Цена: 50,000
- long_score: 65
- short_score: 50

**Этап 1 - Первичное сохранение:**
1. ✅ long_probability = 100% → проходит
2. ✅ type = BUY (не HOLD) → проходит
3. ✅ Нет дубликата за 30 минут → проходит
4. ✅ Сигнал сохраняется в базу

**Этап 2 - Финальная фильтрация:**
1. ✅ Общие критерии: все поля заполнены, цены валидны
2. ✅ TP distance: |50000 - 51000| / 50000 × 100 = 2% ≤ 3% → проходит
3. ✅ BUY критерии:
   - RSI: 52 (48 < 52 < 60) → ✅
   - MACD/ATR: 150/500 = 0.30 ≥ 0.25 → ✅
   - EMA distance: |50000-49500|/500 = 1.0 (0.5 ≤ 1.0 ≤ 1.5) → ✅
   - ATR%: (500/50000)×100 = 1.0% (0.3 ≤ 1.0 ≤ 3.0) → ✅
   - Score diff: 65-50 = 15 (10 ≤ 15 ≤ 20) → ✅

**Результат:** ✅ Сигнал остается в базе данных

---

### Сценарий 2: Сигнал не проходит финальную фильтрацию

**Входные данные:**

- RSI: 65 (слишком высоко)
- MACD Histogram: 100
- ATR: 500
- EMA: 49,500
- Цена: 50,000

**Этап 1 - Первичное сохранение:**
1. ✅ long_probability = 100% → проходит
2. ✅ type = BUY → проходит
3. ✅ Нет дубликата → проходит
4. ✅ Сигнал сохраняется в базу

**Этап 2 - Финальная фильтрация:**
1. ✅ Общие критерии → проходит
2. ✅ TP distance → проходит
3. ❌ BUY критерии:
   - RSI: 65 (65 > 60) → ❌ НЕ ПРОХОДИТ

**Результат:** ❌ Сигнал удаляется из базы данных

## Заключение

Алгоритм сохранения сигналов использует **двухэтапную систему фильтрации**:

1. **Первичная фильтрация** отсеивает невалидные и дублирующиеся сигналы
2. **Финальная фильтрация** применяет строгие критерии качества, основанные на нормализации через ATR

В результате в базе данных остаются только **высококачественные сигналы**, соответствующие всем техническим и статистическим критериям стратегии EMA+RSI+MACD.
